Git的使用方法(以GitHub为例)

2019-10-22 4 min read # 前端开发笔记

此文版权归本站Lucas所有，未经授权禁止任何形式的转载及使用，特此声明！

用户名称示例用ZHANGSAN，邮箱地址用123456@qq.com，两者在实际操作时注意改成自己的用户名和邮箱。

git的安装

在https://git-scm.com/上安装适合系统版本的git，我用的windows 64bit。

git的初始化

• 建立本地仓库，在项目文件夹下或指定文件夹下，右击调出git bash,进行初始化

`$git init`

• 设置账户名，即github的用户ID

$git config -- global user.name "ZHANGSAN"

• 设置邮箱地址，即github的注册和登陆邮箱

$git config --global user.email"123456@qq.com"

本地仓库与github的连接

下面与github连接，有两种方式，一http，二ssh。推荐使用ssh，不会出现远程推送文件过大问题。即错误

error: RPC failed; curl 56 OpenSSL SSL_read: SSL_ERROR_SYSCALL, errfno 10054

• 检测是否生成过key

$cd~/.ssh

如果没有生成过key，会提示No such file or directory。

• 生成账户key

$ssh-keygen -t rsa -C"123456@qq.com"

• 此时会提示在哪个文件下保存key，使用默认回车。

• 此时会再提示设置密码，为了方便不设密码直接回车。

可以在命令行中看到key文件保存为地址，按照文件路径打开文件，复制其中所有文本。

• 在github个人设置中，找到SSH and GPG keys，选择新建SSH。将上面复制的文本粘贴至keys框中，title可随意设置，我设置的是PC名。


• 测试是否配置成功

$ssh -T git@github.com

此时会要求输入key密码，因为刚在没设置密码，密码就为空，然后输入yes。

将本地项目上传到仓库

• 添加远程仓库地址

$git remote add origin git@github.com:ZHANGSAN/yourRepo.gitwison

其中，张三是用户名，yourRepo.gitwison是要上传的仓库，origin后面的可在github具体项目仓库中clone ssh

• 到此算配置完成，可测试是否连接正常

$ git commit -m "first commit"

git的常规四步骤

建议直接在磁盘中新建某文件夹，在文件夹内先进行完前面步骤后，再从远程仓库拉去代码，这样可防止本地也有代码时造成远程和本地有冲突。

• git远程仓库中想要拉取的分支

$ git pull origin ZHANGSAN

• 将修改的内容全部保存

$ git add .

• 提交commit，commit内写本次上传的修改内容

$ git commit -m ""

• 将代码push到远程仓库

$ git push origin ZHANGSAN

git中的其他使用方法与命令

切换ssh与http
• 查看当前remote

$ git remote -v

• 切换至http

$ git remote set-url https://github.com/username/repository.git

• 还可以在.git文件夹中找到config文件，将其中url直接替换成想要更改的地址。

切换分支
• 使用vscode点击左下角分支名称(默认为master)，在输入框中选择分支切换。强烈安利如果使用VSCODE IDE的话安装git easy 插件，省去很多敲命令行的工序，非常方便！

• git命令为$ git checkout dev（dev为分支名称）

将远程覆盖本地
• 我是两台电脑上都写有一部分东西才想起来github托管，一台电脑上传的版本是最新的，直接拉取会有很多冲突，因为选择放弃本地working tree，远程直接覆盖本地。

$ git reset --hard
$ git fetch --all
$ git pull

当提示Already up to date时就覆盖完成。
